# ForenShield AI — SHA-256 해시 작업 정리

> 브랜치: `feature/sha256-hash`  
> 패키지: `com.example.demo` (팀원 업로드 API와 동일)

---

## 변경 요약

팀원 `FileService` 업로드 흐름에 SHA-256 해시 생성을 통합했습니다.

```
업로드 → 원본 저장(uploads/original) → SHA-256 생성 → DB 저장 → 응답
```

## 주요 파일

| 파일 | 역할 |
|------|------|
| `service/HashService.java` | SHA-256 생성 (MessageDigest + HexFormat) |
| `service/FileService.java` | 업로드 + 해시 + Evidence DB 저장 |
| `controller/EvidenceController.java` | `POST /api/evidences/upload` |
| `domain/Evidence.java` | 해시 저장 엔티티 |
| `dto/FileUploadResponse.java` | `evidenceId`, `hashValue` 필드 추가 |

## API 응답 예시

```json
{
  "success": true,
  "message": "파일 업로드 완료",
  "evidenceId": 1,
  "fileName": "sample.mp4",
  "fileSize": 1024,
  "hashAlgorithm": "SHA-256",
  "hashValue": "6e340b9cffb37a989ca544e6bb780a2c78901d3fb33738768511a30617afa01d"
}
```

## Evidence 테이블

| 컬럼 | 타입 | 설명 |
|------|------|------|
| evidence_id | BIGINT | PK |
| file_name | VARCHAR | 원본 파일명 |
| hash_algorithm | VARCHAR(20) | SHA-256 |
| hash_value | VARCHAR(64) | Hex 해시 |
| original_storage_path | TEXT | 로컬 저장 경로 |
| uploaded_at | TIMESTAMP | 업로드 시각 |
